# ETIA官网部署教程（腾讯云COS+CDN）

保姆级部署教程，跟着点按钮就行，全程10分钟。

---

## 第一步：构建静态文件
在本地项目目录执行：
```bash
npm run build
```
构建完成后，会生成 `out/` 文件夹，里面就是所有网站静态文件。

---

## 第二步：创建腾讯云COS存储桶
1. 打开腾讯云控制台：https://console.cloud.tencent.com/cos
2. 点击「创建存储桶」
3. 填写配置：
   - 名称：`etia-website`（随便起，全球唯一就行）
   - 地域：选「亚太地区（上海）」（离你近）
   - 访问权限：**公有读私有写**（必须选这个，不然网站打不开）
   - 其他默认，点确定创建

---

## 第三步：配置静态网站
1. 进入刚创建的存储桶，点左侧「基础配置」→「静态网站」
2. 点「编辑」，开启静态网站
3. 配置：
   - 默认索引文档：`index.html`
   - 默认错误文档：`404.html`
   - 其他默认，点保存
4. 保存后会给你一个「访问节点」域名，记下来，类似 `etia-website-1250000000.cos-website.ap-shanghai.myqcloud.com`

---

## 第四步：上传网站文件
1. 点左侧「文件列表」
2. 点「上传文件」
3. 选择 `out/` 文件夹里的**所有文件和文件夹**（注意是out里面的内容，不是out文件夹本身）
4. 点「上传」，等待上传完成

---

## 第五步：开启CDN加速（必须，不然国内访问慢）
1. 打开CDN控制台：https://console.cloud.tencent.com/cdn
2. 点「添加域名」
3. 配置：
   - 加速域名：填你自己的域名，比如 `www.etia-tech.com`
   - 业务类型：「网页加速」
   - 源站类型：「COS源」
   - 源站地址：选择你刚创建的COS存储桶
   - 回源协议：选「HTTPS」
4. 点下一步，其他默认，点「提交」

---

## 第六步：配置域名解析
1. 去你的域名服务商（GNAME），添加一条CNAME记录
2. 主机记录：`www`
3. 记录类型：`CNAME`
4. 记录值：CDN给你的CNAME地址（在CDN域名管理里能看到，类似 `www.etia-tech.com.cdn.dnsv1.com`）
5. 保存，等10分钟解析生效

---

## 第七步：配置HTTPS证书
1. 在CDN控制台，点你的域名，进入「高级配置」→「HTTPS配置」
2. 点「配置证书」，可以申请腾讯云免费SSL证书
3. 开启「强制HTTPS跳转」，这样访问http会自动跳https

---

## 第八步：配置301重定向（重要，保留百度权重）
在CDN控制台，你的域名→「规则引擎」，添加规则：
1. 旧地址301跳转到新地址，比如：
   - 旧URL：`/products/xxx.html` → 新URL：`/labels/products/xxx/`
   - 旧URL：`/about.html` → 新URL：`/about/`
2. 所有旧URL都要配置301永久重定向，上线前把所有旧链接整理好

---

## 第九步：测试访问
1. 等解析生效后，打开你的域名 `https://www.etia-tech.com`
2. 测试所有页面能不能正常访问
3. 测试手机端适配
4. 打开百度站长平台，提交Sitemap：`https://www.etia-tech.com/sitemap.xml`

---

## 后续更新网站
以后每次更新内容，只需要：
1. 本地执行 `npm run build`
2. 把out文件夹里的文件重新上传到COS，覆盖旧文件
3. CDN会自动缓存更新，5分钟内全国生效

---

## 常见问题
1. **页面刷新404？**
   去COS静态网站配置里，把错误文档改成index.html，因为是单页应用，所有路由都走index.html

2. **访问慢？**
   开启CDN缓存：在CDN缓存配置里，CSS/JS/图片缓存7天，HTML缓存1小时

3. **百度不收录？**
   去百度站长平台验证网站，提交sitemap，开启主动推送
